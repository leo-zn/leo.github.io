java开发之jsp

发表于 2017-11-28 | 分类于 java编程

java 动态页面技术之 jsp

1. jsp 脚本和注释

jsp 脚本:
   1. <%java 代码%>  --> 内部的 java 代码会翻译到 service 方法的内部  
   2. <%=java 变量和表达式%>  --> 会被翻译成 service 方法内部 out.print()
   3. <%!java 代码%> --> 会被翻译成 servlet 的成员变量
注意点: 第一种不可以直接定义方法
  • jsp 注释:
    1. HTML 注释: <!-- 内容 --> jsp 源码、翻译后的 servle、 页面显示 html 源码、 都是可见的
    2. java 注释: // 单行注释 与 /*多行注释*/ 翻译后的 servlet 和 jsp 源码可见
    3. jsp 注释: <%– 内容 –%> 仅 jsp 源码可见

2. jsp运行原理

* jsp 本质就是 servlet
* 第一次访问--> **.jsp --> **.jsp.java --> 编译运行
* 其源码可在 Tomcat 的 work 目录中找到

3. jsp 指令

page 指令属性
   1. language: jsp 脚本可以嵌入的语言种类
   2. pageEncoding: 当前 jsp 文件本身的编码 -- 内部可以包含 contentType
   3. contentType: responce.setContentType(text/html; charset=UTF-8)
   4. session: 是否自动创建 session
   5. import: 导入 java 的包
   6. errorPage:当页面出错后跳转的页面
   7. isErrorPage: 当前页面是一个处理错误的页面

4. include 指令

将一个静态 jsp 中的页面包含到另一个 jsp 页面中 
格式 `<%@ include file=" 被包含的文件地址 "%> `

5. taglib 指定

在 jsp 页面中引入标签库
格式: `<%@ taglib url="标签库地址" prefix="前缀"%>`
  1. jsp 内置/隐式对象 (9个)
out: javax.servlet.jsp.JspWrite  --> 用于页面输出
request: javax.servlet.http.HttpServletRequest --> 得到用户信息
respones: javax.servlet.http.HttpServletReponse --> 服务器向客户端的回应信息
confi:g javax.servlet.ServletConfig  --> 服务器配置, 可以获取初始化参数
session: javax.servlet.http.HttpSession  --> 用来保存用户的信息
application: javax.servlet.ServletContext  --> 所有用户的共享信息
page: java.lang.Object  --> 指当前页面转换后的 Servlet 类的实例
pageContext: javax.servlet.jsp.PageContext  --> JSP 的页面容器
exception: java.lang.Throwable  --> 表示 JSP 页面所发生的异常, 在错误页面财起作用

pageContext 是一个域对象
属性:

1. setAttribute(String name, Object obj)
2. getAttribute(String name)
3. removeAttrbute(String name)
scope 是在指定的域中存储
4. setAttribute(String name, Object obj, init scope)
5. getAttribute(String name, init scope)
6. removeAttrbute(String name, init scope)
7.findAttrbute(String name)
依次在4个作用域中寻找, 从小作用域到大作用域

7. 四大作用域

page 域: 当前 jsp 页面范围
request 域: 一次请求
session 域: 一次会话
application 域: 整个 web 应用

8. 获取其他8 大隐式对象

pageContent.getRequest()
pageContent.geSession()
其余省略

9. jsp 标签(动作)

1. 动态包含: <jsp: include page="被包含的页面"/>
2. 请求转发: <jsp: forward page="要转发的支援"/>

动态包含和静态的区别:
动态包含是运行时载入,编译会产生 多个 .class 的文件, 而静态的会合并成一个文件(编译过后只有一个.class 文件)
