java 字符串StringBuffer

发表于 2016-10-12 | 分类于 javaSE

StringBuffer 类

StringBuffer 线程安全的可变字符序列

StringBuffer 和 String 的区别

String 是一个不可变得字符序列
StringBuffer 是一个可变的字符序列

StringBuffer 的构造方法

public StringBuffer(): 无参构造
public StringBuffer(int capacity): 指定容量的字符串缓冲区对象
public StringBuffer(String str): 指定字符串内容的字符串缓冲区对象

常见功能

append(String str)

把任意数据类型添加到字符创缓冲区里面, 并返回字符串缓区本身

insert(int offset, String str)

在指定位置把任意类型的数据插入到字符串缓冲区里面, 并返回字符串缓冲区本身

deleteCharAt(int index)

删除指定位置的字符, 并返回本身

delete(int start, int end)

删除从指定位置开始 和结束的内容, 并返回本身

replace(int start, int end, String str)

替换指定的字符

reverse()

字符串反转

substring(int start)

从指定位置截取到末尾

substring(int start, int end)

截取指定位置

StringBuffer 和 String 的相互转换

String – StringBuffer

通过构造方法
通过 append() 方法

StringBuffer – String

通过构造方法
通过 toString 方法
通过 subString(0, length) 方法

StringBuffer 和 StringBuilder

StringBuffer  是线程安全的, 效率低
StringBuilder 线程不安全的, 效率高

迭代器模式

发表于 2016-06-12 | 分类于 设计模式

设计模式之迭代器模式

原理分析

  • 迭代器是对集合进行遍历,而每一个集合内部的存储结构是不同的,所有每一个集合存和取都是不一样的, 那么就需要在每一个类中定义 hasNext() 和 next() 方法, 这样做是可以的, 但是会让整个集合体系过于臃肿, 这样做的好处有二:
  • 规定了整个集合体系的遍历方式都是 hasNext() 和 next() 方法
  • 代码由底层内部实现, 使用者不用关注怎么实现, 只需使用即可
